Plan Plataforma Integrada Navistar 2026-2027

Unificación MIOSS-SIV + Remediación BaaN→SAP

Resumen Ejecutivo

Visión Estratégica

Desarrollar una Plataforma Integrada de Ventas para Navistar que consolide las capacidades de MIOSS (exportación) y SIV (mercado mexicano) mientras migra las interfaces críticas de BaaN a SAP, creando un ecosistema tecnológico moderno y unificado.

Objetivos del Programa

  1. Unificación de Sistemas de Ventas: Una plataforma para ambos mercados
  2. Modernización de Interfaces: Migración completa BaaN → SAP
  3. Optimización Operacional: Reducción 60% en costos de mantenimiento
  4. Arquitectura Cloud-Native: Plataforma escalable en Azure
  5. Experiencia de Usuario Superior: Interface moderna y eficiente

Período de Ejecución: Enero 2026 - Enero 2027

Duración Total: 52 semanas (26 sprints de 2 semanas)

Inversión Total: $1.2M - $1.5M USD

Go-Live Coordinado: Enero 2027

ARQUITECTURA DE LA PLATAFORMA INTEGRADA

Componentes de la Solución

1. Plataforma de Ventas Unificada

  • Frontend: Angular 18 con Progressive Web App
  • Backend: .NET 8 con arquitectura modular
  • Base de Datos: Azure SQL con esquemas unificados
  • APIs: RESTful services con OpenAPI documentation

2. Capa de Integración SAP

  • SAP Integration Suite: Conectores nativos SAP
  • Message Bus: Azure Service Bus para comunicación asíncrona
  • Data Transformation: Mapeo y conversión de datos BaaN→SAP
  • API Gateway: Gestión centralizada de interfaces

3. Infraestructura Cloud

  • Azure App Service: Hosting de aplicaciones web
  • Azure SQL Database: Base de datos principal
  • Azure Redis Cache: Caché distribuido para performance
  • Azure Key Vault: Gestión segura de credenciales
  • Application Insights: Monitoreo y analytics

FASES DEL PROGRAMA

FASE 1: FUNDACIÓN E INTEGRACIÓN (Enero - Abril 2026)

Sprints 1-8 | 16 semanas

Objetivos de la Fase

  • Consolidar sistemas MIOSS y SIV
  • Implementar conectividad inicial con SAP
  • Crear framework de interfaces

Entregables Principales

Sprints 1-2 (Enero 2026):
  • Setup de ambiente de desarrollo en Azure
  • Análisis detallado de interfaces BaaN existentes
  • Diseño de arquitectura de integración SAP
  • Setup de repositorios y pipelines CI/CD
Sprints 3-4 (Febrero 2026):
  • Migración de código base MIOSS y SIV a .NET 8
  • Unificación de modelos de datos
  • Implementación de Entity Framework Core
  • Prototipo de interface SAP más crítica
Sprints 5-6 (Marzo 2026):
  • Desarrollo de APIs unificadas de ventas
  • Implementación de Azure Service Bus
  • Mapeo de datos BaaN → SAP (primera iteración)
  • Frontend Angular 18 - módulo de ventas básico
Sprints 7-8 (Abril 2026):
  • Integración completa de 3 interfaces SAP prioritarias
  • Testing de extremo a extremo
  • Validación con usuarios piloto
  • Documentación técnica y de usuario

Estrategia de Coordinación

Interfaces a Remediar

Enfoque de Migración

  1. Análisis Detallado: Mapeo completo de data flows actuales
  2. Desarrollo Paralelo: Construcción de nuevas interfaces mientras mantienen las existentes
  3. Testing Dual: Validación con datos de BaaN y SAP simultáneamente
  4. Migración Gradual: Activación progresiva por interface
  5. Rollback Capability: Posibilidad de revertir a BaaN en caso necesario

Gestión de Dependencias

  • SAP Team Coordination: Alineación semanal con equipo SAP
  • Data Validation: Procesos de validación entre BaaN y SAP
  • Business Continuity: Asegurar 0 interrupciones de negocio
  • Performance Monitoring: Supervisión continua de tiempos de respuesta
